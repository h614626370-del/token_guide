# Product

## Register

product

## Users

Token向云用户、开发者和 AI 工具使用者。他们已经或即将创建 API Key，想快速验证文本模型和图片生成是否可用，比较模型价格，并复制可行的接入配置。

## Product Purpose

本仓库维护 Token向云的官网首页、指南站、模型工作台、价格参考和在线反馈。成功状态是用户能快速理解充值与接入流程，在工作台用最少参数完成一次模型测试，估算常用模型成本，并相信该网关适合真实项目接入。

## Brand Personality

清晰、可靠、克制。界面应该像一个生产工具，而不是营销演示或参数实验室。

## Anti-references

避免过重的参数面板、密集表单、装饰性渐变、夸张玻璃拟态、空洞的 SaaS 卡片堆叠，以及让新用户误以为必须理解所有底层 API 参数才能试用的界面。

## Design Principles

- 先让用户成功发出一次请求，再暴露高级配置。
- 把 API Key、模型、提示词和结果作为主流程，其他参数采用默认值。
- 文本试用像对话工具，图片试用像生成工作台。
- 不在本项目存储历史对话或生成图片，界面文案必须让这一点自然可见。
- 保持指南站和主站入口清楚，避免把工作台做成孤立页面。

## Accessibility & Inclusion

以 WCAG AA 作为默认目标。表单控件需要明确标签、可键盘操作、焦点可见；动效只用于状态反馈，并尊重减少动态效果偏好。
